# Data Shield AI — Дизайн-спецификация

Дата: 2026-06-13
Статус: утверждено, в реализации

## Цель

Локальный офлайн-фильтр приватности между пользователем/приложением и любым
внешним ИИ. Находит конфиденциальные данные в тексте и заменяет их
типизированными стабильными плейсхолдерами **до** отправки во внешнюю модель.
Обработка полностью локальная, оригиналы никогда не пишутся на диск.

## Решения (утверждены пользователем)

1. Движок детекции — **гибрид**: ядро без зависимостей (regex + контрольные
   суммы + энтропия), опциональный ML-плагин (Presidio/spaCy) подключается
   лениво, только если установлен.
2. Покрытие — **РФ + международное**.
3. Режим — **односторонняя редакция** (без обратного восстановления). Оригиналы
   держатся только в памяти и стираются при выходе. Реверс-хранилища нет.
4. Плейсхолдеры — типизированные с номером: `[EMAIL_1]`, `[INN_1]`, `[CARD_2]`.
   Одно и то же значение в рамках одного запроса → один и тот же плейсхолдер.

## Форм-фактор

- Standalone CLI: `python3 -m datashield redact` — чистая stdlib, Python 3.9+,
  без `pip install`.
- Навык Claude Code: `SKILL.md` (автообнаружение).
- Codex: `AGENTS.md` с инструкцией.
- `install.sh` — установка в `~/.claude/skills/data-shield-ai` и подключение
  команды `datashield`.

## Архитектура

```
datashield/
  cli.py            команды: redact · scan · stats · detectors
  api.py            публичный API: redact(), scan(), build_engine()
  engine.py         оркестратор: детекторы → разбор пересечений → маскировка
  masking.py        аллокатор стабильных типизированных плейсхолдеров
  config.py         JSON-конфиг: пороги, паттерны, allowlist, формат
  validators.py     Луна, контрольные цифры ИНН/СНИЛС, IBAN mod-97, энтропия
  detectors/
    base.py         Finding + RegexDetector + KeywordContextDetector
    regex_intl.py   email, телефон, карта, IBAN, IP, MAC
    ru.py           ИНН, СНИЛС, паспорт РФ, телефон РФ
    secrets.py      ключи AWS/OpenAI/Anthropic/GitHub/Google/Slack/Stripe, JWT,
                    приватные ключи, password=…, high-entropy
    registry.py     сборка набора детекторов с учётом конфига
```

## Поток данных

`текст → детекторы (каждый даёт Finding: type, span, value, confidence) →
фильтр по порогу уверенности + allowlist + only/exclude →
разбор пересечений (побеждает выше уверенность, затем длиннее) →
аллокатор стабильных плейсхолдеров → замаскированный текст.`

## Уверенность и пороги

- Порог по умолчанию `min_confidence = 0.7`.
- Высокоточные (всегда маскируются): email 0.98, IBAN 0.95, приватный ключ 0.99,
  распознаваемые токены/ключи 0.95–0.97, JWT 0.9, карта (Луна) 0.9.
- Контекстно-зависимые РФ-ID: ИНН/СНИЛС/паспорт получают буст уверенности при
  наличии ключевого слова рядом («ИНН», «СНИЛС», «паспорт»); без контекста
  слишком общие (паспорт 4+6 цифр) не маскируются по умолчанию.
- Опционально (выключено по умолчанию): high-entropy, ML person/location.

## Валидация (не только regex)

- Карты — алгоритм Луна.
- ИНН — контрольные цифры (10 и 12 знаков).
- СНИЛС — контрольная сумма.
- IBAN — mod-97.
Это удерживает ложные срабатывания низкими: случайный 16-значный номер не
пройдёт Луна.

## Обработка ошибок

- Пересечения матчей — детерминированный разбор.
- Неизвестный тип в `--only/--exclude` → ошибка со списком валидных.
- ML запрошен, но не установлен → предупреждение + работа на ядре.
- Пустой ввод → чистый выход (код 0).

## Приватность отчёта

`--report` пишет только: плейсхолдер, тип, позицию, уверенность, детектор и
**солёный SHA-256 хеш** значения. Сырые значения не сохраняются никогда.

## Тестирование (TDD, stdlib unittest, без зависимостей)

- Валидаторы: известные верные/неверные векторы (Луна, ИНН-10/12, СНИЛС, IBAN).
- Детекторы: позитивные и негативные кейсы (защита от ложных срабатываний).
- Движок: пересечения, стабильная нумерация плейсхолдеров, allowlist, порог.
- CLI: stdin/файл, JSON, коды возврата.
- Приватность: сырые оригиналы не появляются в отчёте.
